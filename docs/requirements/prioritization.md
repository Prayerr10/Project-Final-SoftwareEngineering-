# Prioritization

## Metode Prioritas

Requirement diprioritaskan dengan kategori:

- Must: wajib untuk memenuhi instruksi dosen dan alur utama.
- Should: penting, tetapi dapat dikerjakan setelah fitur inti.
- Could: tambahan jika waktu cukup.
- Won't: tidak dikerjakan pada scope project ini.
